负边距

在文档流中，元素的最终边界是由margin决定的，margin为负的时候就相当于元素的边界向里收，文档流认的只是这个边界，不会管你实际的尺寸是多少。

清浮动

1.给浮动元素父级加float — 不实用
2.空标签清浮动

<style>
  .clear{ clear:both; }
</style>

<div class="clear"></div>

3.在父级加overflow: hidden清浮动

.clear{
  overflow: hidden;
  zoom:1;
}

4.after伪类清浮动

.clear{
  zoom:1;
}
.clear:after{
  visibility: hidden; /* Firefox < 3.5 */
  content: "";
  clear: both;
  display: block;
}
  • Bootstrap3的清除是这样的：
/**
 * 现代浏览器
 * 1. content的空格内容是为了避免Opera浏览器bug，但是它会导致空格内容出现在元素顶部和底部。
 * 2. 当使用 `before` 包含子元素的上边距时，必须使用 `table` 而不使用 `block`，可以防止上边距折叠。
 */
.cf:before,
.cf:after {
  content: " "; /* 1 */
  display: table; /* 2 */
}

.cf:after {
  clear: both;
}

/**
 * For IE 6/7 only
 * 触发hasLayout并清除浮动
 */
.cf {
  *zoom: 1;
}

支持：Firefox 3.5 +，Safari 4 +，Chrome，Opera 9 +，IE 6+
资料：clearfix hack

  • Bootstrap4的清除是这样的：
.clearfix::after{
  display: block;
  clear: both;
  content: "";
}
