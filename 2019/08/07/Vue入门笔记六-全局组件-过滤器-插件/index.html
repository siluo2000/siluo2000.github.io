Vue入门笔记六(全局组件 过滤器 插件 变量)

发表于 2019-08-07 | 分类于 Vue | | 阅读次数:

《Vue.js项目实战》

全局组件

1.在components文件夹创建Loading.vue：

<template>
  <div class="loading">
    <div></div>
  </div>
</template>

2.在main.js旁边创建一个global-components.js，然后使用Vue.component()方法全局地注册Loading组件：

import Vue from "vue"
import Loading from "./components/Loading"

Vue.component("Loading", Loading)

3.然后，在main.js中导入global-components.js：

import './global-components'

现在可以全局使用<Loading />了！

全局过滤器

1.以momentjs为例，首先安装moment：

npm install -S moment

2.在main.js文件旁边创建一个新的filters.js，它包含date过滤器：

import moment from "moment"

export function date (value) {
  return moment(value).format('L')
}

3.在main.js中导入filters.js并用一个循环注册它们：

import * as filters from './filters'

for(const key in filters) {
  Vue.filter(key, filters[key])
}

现在完成注册全局过滤器了！

  • 在组件中使用：
<div class="info">
  Created on <strong>{{ ticket.date | date }}</strong>
</div>

全局插件

用自己的插件扩展Vue：

创建一个插件

（1）在src文件夹中创建一个plugins文件夹
（2）在plugins文件夹中创建一个fetch.js
（3）让我们通过导出一个带有install对象的方法创建一个插件：

export default {
  install (Vue) {
    console.log('Installed!')
  }
}

（4）在main.js中导入这个插件，然后像我们为vue-router所做的那样调用Vue.use()方法：

import VueFetch from './plugins/fetch'

Vue.use(VueFetch)

插件选项

我们可以使用options参数配置插件。
（1）编辑install方法，添加options参数：

export default {
  install (Vue, options) {
    console.log('Installed!', options)
  }
}

（2）添加一个baseURL属性到配置中：

Vue.use(VueFetch, {
  baseURL: 'http://localhost:3000/',
})

（3）将baseURL存储到一个变量中，稍后使用：

let baseURL
export default {
  install (Vue, options) {
    console.log('Installed!', options)

    baseURL = options.baseURL
  }
}

$fetch方法

（1）在fetch.js中使用fetch方法实现$fetch方法：

export async function $fetch (url) {
  const response = await fetch(`${baseURL}${url}`)
  if(response.ok) {
    const data = await response.json()
    return data
  } else {
    const error = new Error('error')
    throw error
  }
}

（2）添加到Vue的原型，使它在所有组件中都可用：

export default {
  install (Vue, options) {
    console.log('Installed!', options)

    baseURL = options.baseURL

    Vue.prototype.$fetch = $fetch
  }
}

（3）然后在组件中使用$fetch方法，比如：

try {
  this.question = await this.$fetch('question')
} catch (e) {
  this.error = e
}
改进fetch插件

目前，我们的$fetch方法只能对服务器发出GET请求。

（1）在plugins/fetch.js文件中添加options参数：

export async function $fetch(url, options) {
  // ...
}

options参数是fetch方法的可选参数，允许我们更改不同的参数，例如使用的HTTP方法、请求主体等。

（2）在$fetch方法头部设置option参数的默认值：

const finalOptions = Object.assign({}, {
  headers: {
    'Content-Type': 'application/json'  // 始终在请求主体中发送JSON
  },
  credentials: 'include'                // 包含用户登录所需的授权令牌
}, options)

（3）接下来，将新选项添加到浏览器fetch方法：

const response = await fetch(`${baseURL}${url}`, finalOptions)

（4）另外，服务器总是将错误作为文本发送，所以可以捕获它们并显示给用户：

if(response.ok) {
  const data = await response.json()
  return data
} else {
  const message = await response.text()
  const error = new Error(message)
  error.response = response
  throw error
}
  • 在组件中使用：
async signup () {
  await this.$fetch('signup', {
    method: 'POST',
    body: JSON.stringify({
      username: this.username,
      password: this.password,
      email: this.email
    })
  })
  this.mode = 'login'
}

（5）会话过期

服务器始终返回403错误。

1.在修改请求方法（/plugin/fetch.js）前，需要导入state和路由器：

import state from "../state"
import router from "../router"

2.在fetch.js响应处理中添加一个条件分支：

if(response.ok) {
  // ...
}else if(response.status === 403){
  // 如果会话不在有效
  // 我们登出
  state.user = null
  // 如果这个路由是私有的
  // 我们跳转到登录页面
  if(router.currentRoute.matched.some(r => r.meta.private)) {
    router.replace({name: 'login', params: {
      wantedRoute: router.currentRoute.fullPath
    }})
  }
} else {
  // ...
}

现在这个插件完成了！

import state from "../state"
import router from "../router"
let baseUrl

export async function $fetch (url, options) {
  const finalOptions = Object.assign({}, {
    headers: {
      'Content-Type': 'application/json'
    },
    credentials: 'include'
  }, options)
  const response = await fetch(`${baseUrl}${url}`, finalOptions)
  if(response.ok) {
    const data = await response.json()
    return data
  }else if(response.status === 403){
    // 如果会话不在有效
    // 我们登出
    state.user = null
    // 如果这个路由是私有的
    // 我们跳转到登录页面
    if(router.currentRoute.matched.some(r => r.meta.private)) {
      router.replace({name: 'login', params: {
        wantedRoute: router.currentRoute.fullPath
      }})
    }
  } else {
    const message = await response.text()
    const error = new Error(message)
    error.response = response
    throw error
  }
}

export default {
  install (Vue, options) {
    baseUrl = options.baseUrl

    Vue.prototype.$fetch = $fetch
  }
}

全局变量/对象

将用户存储在一个集中式state里

（1）在main.js旁边创建state.js，用于导出state对象：

export default {
  user: null
}

（2）在main.js导入state：

import state from "./state"

（3）然后，将其用作根实例的数据，以便使其成为响应式的：

new Vue({
  el: '#app',
  data: state,
  render: h => h(AppLayout),
  router
});

$state原型对象

（1）在plugins文件夹中，创建一个导出插件state.js：

export default {
  install (Vue, state) {
    Object.defineProperty(Vue.prototype, '$state', {
      get: () => state,
    })
  }
}

（2）在main.js中导入插件：

import VueState from './plugins/state'

Vue.use(VueState, state)

现在可以在组件中使用$state来访问全局状态了！

  • 实例：
<template v-if="$state.user">
  <!-- ... -->
</template>

<script>
export default {
  created() {
    console.log(this.$state)
  }
}
</script>
