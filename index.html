NODE安装笔记

2019-08-10

卸载node和npm

在安装nvm前要先卸载已经安装的node和npm。

命令行卸载的方法没看懂。可以找到自带的卸载程序卸载node。

安装nvm

1.NVM下载
2.下载完成后，将内容解压到C:\dev\nvm （需为全英文路径）
3.新建settings.txt文件：

root: C:\dev\nvm
path: C:\dev\nodejs
arch: 64
proxy: none
node_mirror: http://npm.taobao.org/mirrors/node/
npm_mirror: https://npm.taobao.org/mirrors/npm/

4.配置环境变量

  • window + r 打开运行面板，然后输入 : sysdm.cpl，打开系统属性面板
  • 在系统属性面板中切换到高级面板，点击下方的环境变量按钮，打开环境变量面板
  • 新建系统变量如下：
NVM_HOME: C:\dev\nvm
NVM_SYMLINK: C:\dev\nodejs
  • 在系统变量PATH中新增 %NVM_HOME%;%NVM_SYMLINK%;

5.打开cmd，检查是否配置成功

set NVM_HOME
set NVM_SYMLINK
nvm -v

安装node

命令：

# 查看所有版本
nvm ls

# 查看可安装版本
nvm ls-remote

# 安装最新版本
nvm install latest

# 安装随意版本
nvm install 4.6.0

# 切换node版本
nvm use 4.6.0

# 安装稳定版本
nvm install stable

# 查看当前版本
nvm current

配置node全局模块路径

  • 方法一：
    设置到自己想要放的位置：

打开：nodejs安装目录/node_modules/npm/.npmrc这个文件，修改里面的路径

prefix = E:\nodejs\node_global_modules
  • 方法二：

进入cmd，输入如下代码：

# 查看npm配置
npm config ls  

# 配置prefix修改node的全局模块安装路径
npm config set prefix E:\nodejs\node_global_modules

预渲染 Prerendering

prerender-spa-plugin

npm i -D prerender-spa-plugin

在webpack.config.js中：

// Base Usage
const path = require('path')
const PrerenderSPAPlugin = require('prerender-spa-plugin')
const Renderer = PrerenderSPAPlugin.PuppeteerRenderer

module.exports = {
  plugins: [
    // ...
    new PrerenderSPAPlugin({
      staticDir: path.join(__dirname, 'dist'), //你要存放的静态资源目录
      routes: ['/', '/faq'], //设置要对哪个文件进行SEO
      //如果没有这个配置不会进行预编译
      renderer: new Renderer({
        inject: {
          foo: 'bar',
        },
        headless: true,
        //在项目的入口中使用document.dispatchEvent(new Event('render-event'))
        //这个是你生命的事件名
        rendererAfterDocumentEvent: 'render-event'
      })
    })
  ]
}
  • 应该要在生产环境下使用，然后输出预渲染的HTML，方便搜索引擎爬虫抓取工具可以直接查看预渲染的内容。

将样式提取到CSS文件

安装包：

npm i -D extract-text-webpack-plugin

在webpack.config.js导入并配置：

const ExtractTextPlugin = require('extract-text-webpack-plugin')

module.exports = {
  // ...
  plugins: [
    // ...
    new ExtractTextPlugin({
      filename: 'common.[chunkhash].css',
    }),
  ]
}

新的npm脚本

1.安装包：

npm i -D rimraf

2.为客户端bundle和服务器bundle各添加一个build脚本：

{
  // ...
  "build:client": "cross-env NODE_ENV=production webpack --progress --hide-modules --config webpack/client.js",
  "build:server": "cross-env NODE_ENV=production webpack --progress --hide-modules --config webpack/server.js"
}

3.新建一个build脚本，用来清空dist文件夹并运行build:client和build:server这两个脚本：

"build": "rimraf dist && npm run build:client && npm run build:server"

4.添加最后一个名为start的脚本，它会以生产模式运行express服务器：

"start": "cross-env NODE_ENV=production node server"

5.运行构建：

npm run build

6.启动express服务器：

npm start

Vue入门笔记六(全局组件 过滤器 插件 变量)

2019-08-07

《Vue.js项目实战》

全局组件

1.在components文件夹创建Loading.vue：

<template>
  <div class="loading">
    <div></div>
  </div>
</template>

2.在main.js旁边创建一个global-components.js，然后使用Vue.component()方法全局地注册Loading组件：

import Vue from "vue"
import Loading from "./components/Loading"

Vue.component("Loading", Loading)

3.然后，在main.js中导入global-components.js：

import './global-components'

现在可以全局使用<Loading />了！

Vue入门笔记五(常用插件)

2019-08-07

《Vue.js项目实战》

axios

1.安装包：

npm install axios -S

2.在src/utils文件夹中创建http.js：

import axios from 'axios'

const http = axios.create({
  baseURL: 'http://localhost:3000/',
})

export function install (Vue) {
  Vue.prototype.$http = http
}

export default http
  • 在src/plugins.js中导入和注册http：
import Http from './utils/http'

Vue.use(Http)

3.在vuexStore中使用：

import http from '../utils/http'

let result = await http.get(`items/${id}`)
let result = await http.get(`comments?itemId=${id}`)
let result = await http.post('comments', data)

在组件中使用：

const result = await this.$http.post('comments', params)
